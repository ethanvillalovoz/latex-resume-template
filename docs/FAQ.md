# FAQ

## Which file should I edit first?

Use `src/starter-resume.tex` for a blank resume and `src/cv/starter-cv.tex` for a blank academic CV. The files `src/resume.tex` and `src/cv/cv.tex` are polished examples with real content.

## Can I use this in Overleaf?

Yes. Upload the repository or the `src/` folder to Overleaf. For the academic CV, keep `src/cv/res.cls` in the same folder as `cv.tex` or `starter-cv.tex`.

## Which compiler should I use?

Use XeLaTeX when possible. The GitHub Actions workflow compiles the documents with `latexmk` and XeLaTeX.

## Why are the links colored?

The templates use a conservative blue link color so PDF readers can tell which text is clickable while keeping the document professional and printable.

## Is the resume ATS-friendly?

The resume avoids image-based text and decorative layouts that commonly break PDF parsing. It uses semantic text, simple section headings, and machine-readable output settings.

## Should I keep the publications section?

Keep it if you have peer-reviewed publications, preprints, or research artifacts worth surfacing. If not, remove the section and use the space for experience, projects, or technical skills.

## How do I make the resume one page?

Prioritize recent experience, strong projects, and technical skills. Trim weak bullets before shrinking margins or font size. A good one-page resume should feel selective, not cramped.

## Why is there both a resume and an academic CV?

The resume is for technical recruiting and should usually be one page. The CV is for research, academic, fellowship, lab, and graduate-school contexts where publications, advisors, teaching, awards, and outreach matter.

## Can I remove Ethan's content?

Yes. The starter files are designed for that. Replace the placeholders with your own information and use the example files as formatting references.
